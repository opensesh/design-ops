# Decision Framework

Apply structured decision-making frameworks to complex choices.

## Purpose

Help make better decisions by applying proven frameworks that ensure all perspectives are considered, accountability is clear, and decisions are documented for future reference.

## When to Activate

Use this skill when:
- Facing a decision with multiple stakeholders
- Choosing between competing options
- Needing to document decision rationale
- Decisions keep getting stuck or revisited

---

## Available Frameworks

### RAPID Framework
**Best for:** Organizational decisions with multiple stakeholders
- **R**ecommend: Who proposes the decision
- **A**gree: Who must agree before proceeding
- **P**erform: Who implements the decision
- **I**nput: Who provides input but doesn't decide
- **D**ecide: Who has final authority

### DACI Framework
**Best for:** Project decisions, cross-functional teams
- **D**river: Owns the process, ensures decision is made
- **A**pprover: Final decision maker
- **C**ontributors: Provide input and expertise
- **I**nformed: Need to know the outcome

### Weighted Decision Matrix
**Best for:** Comparing multiple options against criteria
- List criteria and weight by importance
- Score each option against each criterion
- Calculate weighted scores

### Six Thinking Hats
**Best for:** Exploring decisions from multiple angles
- White: Facts and data
- Red: Emotions and intuition
- Black: Risks and problems
- Yellow: Benefits and optimism
- Green: Creative alternatives
- Blue: Process and meta-thinking

### Reversibility Test
**Best for:** Quick assessment of decision stakes
- One-way door: Hard to reverse, decide carefully
- Two-way door: Easy to reverse, decide quickly

## Decision Process

### Step 1: Frame the Decision

Clarify what's being decided:

```
## Decision: [Clear question to answer]

**Context:** [Why this decision is needed now]
**Constraints:** [Non-negotiables, limits]
**Timeline:** [When decision must be made]
**Stakes:** [One-way door or two-way door?]
```

### Step 2: Identify Options

List viable options:

```
### Options

1. **[Option A]**
   - Description: [What this means]
   - Pros: [Benefits]
   - Cons: [Drawbacks]

2. **[Option B]**
   - Description: [What this means]
   - Pros: [Benefits]
   - Cons: [Drawbacks]

3. **[Option C: Status quo]**
   - Description: Do nothing / continue current state
   - Pros: [Benefits]
   - Cons: [Drawbacks]
```

### Step 3: Apply Framework

Choose and apply the appropriate framework:

**For RAPID/DACI:**
```
### Decision Roles (DACI)

| Role | Person | Responsibility |
|------|--------|----------------|
| Driver | [Name] | Owns the decision process |
| Approver | [Name] | Makes final call |
| Contributors | [Names] | Provide input |
| Informed | [Names] | Need to know outcome |
```

**For Weighted Matrix:**
```
### Decision Matrix

| Criteria | Weight | Option A | Option B | Option C |
|----------|--------|----------|----------|----------|
| [Criterion 1] | [1-5] | [1-5] | [1-5] | [1-5] |
| [Criterion 2] | [1-5] | [1-5] | [1-5] | [1-5] |
| [Criterion 3] | [1-5] | [1-5] | [1-5] | [1-5] |
| **Weighted Total** | | [sum] | [sum] | [sum] |
```

**For Six Hats:**
```
### Perspectives

**White Hat (Facts):**
- [What do we know for certain?]

**Red Hat (Feelings):**
- [What's the gut reaction?]

**Black Hat (Risks):**
- [What could go wrong?]

**Yellow Hat (Benefits):**
- [What's the upside?]

**Green Hat (Alternatives):**
- [What other options exist?]

**Blue Hat (Process):**
- [What's the best way to decide?]
```

### Step 4: Document Decision

```
## Decision Record

**Decision:** [What was decided]
**Date:** [Date]
**Decider:** [Who made the call]

### Context
[Why this decision was needed]

### Options Considered
1. [Option 1] - [Why rejected/chosen]
2. [Option 2] - [Why rejected/chosen]
3. [Option 3] - [Why rejected/chosen]

### Rationale
[Why this option was selected]

### Expected Outcomes
- [What we expect to happen]

### Risks Accepted
- [Known risks we're accepting]

### Review Trigger
[When/how we'll know if this was wrong]

### Next Steps
- [ ] [Action item] - Owner: [Name] - Due: [Date]
```

## Quick Decision Filters

For fast triage:

1. **Is this reversible?** → If yes, decide quickly
2. **Who has the context?** → They should decide
3. **What's the cost of delay?** → High cost = decide now
4. **Do we have enough info?** → If no, what's the minimum needed?

## Common Decision Traps

| Trap | Symptom | Antidote |
|------|---------|----------|
| Analysis paralysis | Can't stop researching | Set a decision deadline |
| HiPPO (highest paid person's opinion) | Defers to seniority | Use structured framework |
| Sunk cost | "We've already invested..." | Focus on future value only |
| Confirmation bias | Only seeing supporting evidence | Devil's advocate review |
| Groupthink | Everyone agrees too easily | Require written opinions first |
| Status quo bias | Defaulting to current state | Explicitly evaluate "do nothing" |

---

## When to Use Which Framework

| Situation | Recommended Framework |
|-----------|-----------------------|
| Multiple stakeholders, organizational | RAPID |
| Project team decision | DACI |
| Comparing 3+ options with clear criteria | Weighted Matrix |
| Complex, emotional, or novel decision | Six Thinking Hats |
| Quick assessment of importance | Reversibility Test |
| Any decision | Document it |
