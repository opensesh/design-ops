# /install-skills

Onboarding command that sets up a personalized "mini team of agents" for the user's workflows.

## Trigger

User runs `/install-skills` after adding the company-skills plugin. Works in:
- Claude Desktop (Code tab)
- Claude Code CLI (terminal)

---

## Pre-Flight Check

Before starting the interview, verify setup:

1. **Check if config already exists** at `~/.claude/skills-config.yaml`
   - If exists, ask: "You already have a config. Want to update it or start fresh?"
   - Options: `Update existing` | `Start fresh`

2. **Detect available MCP servers** by checking what's configured
   - Note which MCPs are already connected
   - This informs the interview and final summary

---

## Interview Flow

Keep this light — about 2 minutes total. Use the AskUserQuestion tool for structured input.

### Welcome Message

```
Welcome to Company Skills!

You're setting up a personal "agent team" that helps with:
- Morning briefings and task prioritization
- Meeting prep and follow-ups
- Critical thinking and decision support
- Content creation for social platforms

This takes about 2 minutes. I'll ask a few questions to personalize your experience.
```

### Question 1: Your Tools

**Purpose:** Understand what tools they use daily (helps contextualize commands)

```
What tools do you use for work?
(List whatever you use — Calendar, Notion, Slack, email, project tools, etc.)
```

- Accept freeform text
- Parse into a list for the config
- Common tools to recognize: Google Calendar, Notion, Slack, Asana, Linear, Jira, Gmail, Outlook, Figma, GitHub, etc.

### Question 2: Workflow Rhythm

**Purpose:** Understand when they want help (daily, weekly, as-needed)

Present checkboxes:

```
How do you want help organized?

Daily workflows:
[ ] Morning brief — start the day with priorities
[ ] Meeting prep — get ready for upcoming meetings

Weekly workflows:
[ ] Weekly recap — reflect on the week, plan the next

As-needed tools:
[ ] Decision support — challenge assumptions, red-team thinking
[ ] Research — analyze websites, competitors
[ ] Content — social posts for LinkedIn, Instagram, Substack
```

- Let them select multiple
- Map selections to specific commands

### Question 3: Quick MCP Check

**Purpose:** Show what's connected and offer to set up missing pieces

```
Checking your connected tools...
```

Then display results:

```
✅ GitHub — ready (code search, PR reviews)
✅ Supabase — ready (database queries)
⚠️ Google Calendar — not connected (helps with daily briefs)
⚠️ Notion — not connected (helps with task management)

Would you like to set up any missing tools now?
```

- Options: `Set up [Tool Name]` | `Skip for now`
- If they choose to set up, reference the MCP education flow in `mcp-setup/`
- If they skip, note that commands will work in degraded mode

## Generate Config

After the interview, write to `~/.claude/skills-config.yaml`:

```yaml
# Company Skills Configuration
# Generated by /install-skills on [DATE]

version: 1.0
created: [ISO DATE]

# Tools the user works with
tools_used:
  - Google Calendar
  - Notion
  - Slack
  - GitHub

# Connected MCP servers with their capabilities
mcp_connected:
  - name: github
    capabilities: [code_search, pull_requests, issues]
  - name: supabase
    capabilities: [database, auth, storage]

# Workflow preferences
workflows:
  daily:
    - morning_brief
    - meeting_prep
  weekly:
    - weekly_recap
  as_needed:
    - devils_advocate
    - site_analysis
    - social_post

# Optional brand customization (populated if user provides)
# brand:
#   name: "Company Name"
#   voice: "friendly, technical, concise"
```

**Important:** Only include what's actively selected — no empty arrays or false flags.

## Error Handling

**If config directory doesn't exist:**
```bash
# Mac/Linux
mkdir -p ~/.claude

# Windows
mkdir %USERPROFILE%\.claude
```

**If write fails:**
```
I couldn't write the config file. Please check permissions for ~/.claude/

You can create it manually:
1. Run: mkdir -p ~/.claude (Mac/Linux) or mkdir %USERPROFILE%\.claude (Windows)
2. Run /install-skills again

Still having trouble? See the troubleshooting guide: references/troubleshooting.md
```

**If no selections made:**
- Enable all commands by default
- Note: "I've enabled everything — you can run /customize later to narrow down."
